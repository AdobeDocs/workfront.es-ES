# Referencia de calidad de escritura

Reglas detalladas para la aptitud de `writing-quality`. El archivo SKILL.md cubre las reglas más comunes; este archivo proporciona la referencia completa para la puntuación, terminología, ejemplos de uso de mayúsculas y formato de procedimiento.

---

## Puntuación

### Períodos

- Use puntos en una lista si los elementos completan una frase (&quot;En el menú X, puede:&quot;)
- Use puntos en una lista si algún elemento es una frase completa
- Los puntos no deben tener formato con el contenido principal (Correcto: Haga clic en **Listo**. / Incorrecto: Haga clic en **Listo.**)
- Usar puntos en las extensiones de archivo (.pdf)

### Comas

- La coma de Oxford siempre: &quot;motivo, medio y oportunidad&quot;
- Coma después de la frase introductoria: &quot;Para abrir un documento, haga clic en Abrir&quot;.
- Coma después del año en una fecha a mitad de la frase: &quot;El 2 de enero de 2016, número de...&quot;
- Coma antes de &quot;then&quot; en los procedimientos: &quot;Haga clic en **Configuración** y, a continuación, haga clic en **Interfaz**.&quot;
- Sin comas solo mes-año: &quot;Enero de 2016&quot;

### Apóstrofes

- Uso en contracciones: no, son, es...
- Singular posesive: add &#39;s (ordenador de Jen)
- Final posesivo plural en s: añadir solo &#39; (el equipo de los Jones)
- Distinguir &quot;es&quot; (es) de &quot;su&quot; (perteneciente a él)

### Dos puntos

- Minúsculas después de dos puntos a menos que vayan seguidas de comillas directas, sustantivos propios o listas con viñetas
- Se utiliza al final de una frase o fragmento que introduce una lista
- No hay dos puntos después de encabezados de procedimiento
- No hay dos puntos para introducir elementos visuales o segmentos de código

### Punto y coma

- Conectar dos cláusulas sin una conjunción
- Frases separadas que contienen su propia puntuación
- No ponga en negrita un punto y coma después del texto en negrita en los procedimientos
- No incluir en un hipervínculo

### Comillas

- Hágalos referencia como &quot;comillas&quot; (no como &quot;comillas&quot;)
- Las comillas de cierre no incluyen comas ni puntos
- La colocación del signo de interrogación depende de si la cita o la oración es la pregunta

### Guiones

- Evite estructuras que requieran guiones cuando sea posible
- Dividir modificadores compuestos antes de un sustantivo: &quot;memoria de solo lectura&quot;
- Dividir cuando un modificador es un participio: &quot;disco protegido contra copia&quot;
- Dividir cuando un modificador es un número: &quot;Fuente de 12 puntos&quot;
- Sin guiones en las combinaciones de teclas: utilice + (Ctrl+I)

### Guiones elásticos

Use con moderación. Considere reestructurar la sentencia en su lugar.

### Guiones finalizados

- Intervalos: 2005-2013
- Signo menos o números negativos
- Conectar el adjetivo compuesto a otra palabra: &quot;opciones de tipo cuadro de diálogo&quot;

### Elipses

- Código o sintaxis omitidos
- Indicación de interfaz de usuario de que aparecerá un cuadro de diálogo
- Texto omitido entre comillas (añada un punto si reemplaza el final de la frase)

### Barra diagonal

- Sin barra para las opciones: use &quot;o&quot; (&quot;él o ella&quot; no &quot;él/ella&quot;)
- Se utiliza para combinaciones: interruptor de encendido / apagado
- Se utiliza para direcciones y fracciones de Internet

## Ejemplos de uso de mayúsculas

### Términos de Workfront: referencia directa e indirecta

| Término | Directo (mayúsculas) | Indirecto (minúsculas) |
|------|---------------------|----------------------|
| Fecha planificada de finalización | &quot;Complete el campo **Fecha planificada de finalización**&quot;. | &quot;Cada tarea debe tener una fecha planificada de finalización&quot;. |
| Plantilla de diseño | &quot;En la configuración, vaya a **Plantillas de diseño**&quot;. | &quot;Asignar una plantilla de diseño a cada usuario.&quot; |
| Administrador del grupo | &quot;Busque el campo **Administrador del grupo**.&quot; | &quot;Póngase en contacto con el administrador del grupo&quot;. |
| Formulario personalizado | &quot;Adjunte el **formulario personalizado** al proyecto.&quot; | &quot;Puede crear un formulario personalizado para cualquier objeto&quot;. |

### Términos comunes de Workfront (utilice mayúsculas cuando haga referencia a la IU)

- Formulario personalizado
- Duración
- Horas planificadas
- Fecha planificada de finalización
- Tipo de hora
- Administrador de grupo / Administrador del sistema
- Administrador de Workfront (la persona, siempre &quot;administrador&quot; en minúsculas)
- Notificaciones de recordatorio
- Plantilla de diseño
- Filtrar, Ver, Agrupación (en el contexto de Filtros, Vistas y Agrupaciones)
- Planificador de recursos (planes)

### Botones

- Seguir mayúsculas de la IU
- Si el botón está en mayúsculas en la interfaz de usuario, escríbalo en mayúsculas

## Terminología

### Elementos de IU

| Elemento | Término correcto |
|---------|-------------|
| Lista de opciones contraíbles | Menú (añada la lista desplegable para una mayor claridad cuando sea necesario) |
| Comandos no disponibles | &quot;no disponible&quot; (nunca &quot;atenuado&quot;, &quot;deshabilitado&quot; o &quot;atenuado&quot;) |
| Emergente con campos de formulario | Cuadro de diálogo |
| Botones sin nombre | Utilizar el nombre de la información sobre herramientas del botón |
| Mensajes de error/notificaciones de mensajes de texto | &quot;Mensaje&quot; (&quot;Aparece un mensaje que le notifica...&quot;) |
| Entrada de texto | Campo |

### Funciones

| Contexto | Término |
|---------|------|
| La función en Workfront | Administrador del sistema |
| La persona que administra Workfront | Administrador de Workfront |

### Sobrescribir frente a anular

Estas son palabras diferentes con diferentes significados: use la correcta para el contexto.

### Extensiones de archivo

Utilice siempre la extensión de archivo, no las siglas: &quot;Cargar un archivo .pdf&quot; (no &quot;Cargar un PDF&quot;).

## Formato del procedimiento

### Introducciones

Incluya solo cuando sea necesario. Formatos preferidos (en orden):

1. Frase infinita + dos puntos: &quot;Para crear una contraseña temporal para un usuario en su sistema:&quot;
2. Frase completa + punto: &quot;Debe crear contraseñas temporales para los usuarios de su sistema&quot;.
3. Frase infinita + &quot;seguir estos pasos&quot; + dos puntos: &quot;Para crear una contraseña temporal para un usuario, siga estos pasos:&quot;

### Convenciones de pasos

- Utilice listas numeradas para todos los procedimientos, incluso los de un solo paso
- Acciones en negrita en las que se puede hacer clic y elementos de IU dentro de pasos
- Use verbos de acción específicos: &quot;escriba&quot;, &quot;seleccionar&quot;, &quot;hacer clic&quot;, evite &quot;establecer&quot; o &quot;especificar&quot;
- Describir las respuestas del sistema antes o inmediatamente después de que se produzcan

Ejemplo de ubicación de respuesta del sistema:

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### Pasos condicionales y opcionales

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### Subpasos y listas de campos

Introduzca con un comando obligatorio (Configurar, Especificar, Hacer) y, a continuación, campos de lista:

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### Varios métodos

Documente primero la forma más directa:

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### Corchetes angulares derechos

- Mostrar navegación de menú/pestaña: &quot;Haz clic en **Correo electrónico** > **Notificaciones**&quot;
- Nombres de los botones en negrita, no corchetes
- Espacios entre corchetes
- No utilizar para menú principal: utilizar el fragmento de menú principal

## Referencia en negrita y cursiva

### Negrita — Se utiliza para:

| Contexto | En pasos | Pasos exteriores |
|---------|----------|---------------|
| Botones o vínculos en los que puede hacer clic | Negrita | No te atrevas |
| Nombres de menú | Negrita | No te atrevas |
| Nombres de iconos | Negrita | No te atrevas |
| Nombres del cuadro de diálogo | Negrita | No te atrevas |
| Nombres de página | Negrita | No te atrevas |
| Nombres de campo en listas de campos | Negrita + dos puntos | No te atrevas |

### Cursiva — Se utiliza para:

- Texto que el usuario debe escribir: &quot;Escriba *my.workfront.com* en el cuadro Dirección URL&quot;
- Nunca para enfatizar — reestructura la oración en su lugar

## Notas, sugerencias y advertencias: orientación detallada

### Cuándo usar cada

**Nota** (`>[!NOTE]`):
- Información que no se ajusta al contexto del párrafo
- Evite la frustración del usuario o la pérdida de tiempo (&quot;Debe tener una licencia de equipo para completar esta tarea&quot;).
- Información de versiones o compatibilidad (&quot;Esta funcionalidad solo está disponible en el visualizador de pruebas heredadas&quot;).

**Sugerencia** (`>[!TIP]`):
- Sugerencias que facilitan la vida del usuario, pero que no son cruciales
- Ejemplo: &quot;Puede compartir varios documentos al mismo tiempo manteniendo pulsada la tecla Mayús&quot;.

**Importante** (`>[!IMPORTANT]`):
- Información vital para el paso o procedimiento

**Advertencia** (`>[!WARNING]`):
- Alertar al usuario sobre la posible pérdida de datos

### Reglas

- Uno por sección como máximo
- Nunca apilar varias notas/sugerencias/advertencias
- Combinar notas relacionadas en una con una lista con viñetas
- No use notas para anunciar nuevas características, rediseñe el texto del artículo

## Patrones de referencia cruzada

| Situación | Patrón |
|-----------|---------|
| Después de proporcionar información | &quot;Para obtener más información, consulte [Nombre del artículo](link).&quot; |
| Sin información previa | &quot;Para obtener más información, consulte [Nombre del artículo](link).&quot; |
| A una sección de otro artículo | &quot;Para obtener más información, consulte [Sección] en [Nombre del artículo](link).&quot; |
| A una sección del mismo artículo | &quot;Para obtener más información, consulte [Sección] en este artículo.&quot; |

Ubicación:
- En los párrafos: en línea
- En pasos del procedimiento: nueva línea después del paso
- Varias referencias: lista con viñetas

## Características de calidad DQTI

### Fácil de usar

| Característico | Preguntas clave |
|---------------|---------------|
| Orientación de tarea | ¿Se centra en lo que el usuario puede hacer? ¿Es el encabezamiento un verbo imperativo? |
| Precisión | ¿Coincide exactamente la terminología con la interfaz de usuario? ¿Son correctos los detalles técnicos? |
| Integridad | ¿Hay suficiente información para actuar? ¿Hay información innecesaria que eliminar? |

### Fácil de entender

| Característico | Preguntas clave |
|---------------|---------------|
| Claridad | ¿Cada frase es una idea? ¿Las palabras son simples y directas? |
| Concreción | ¿Hay ejemplos específicos? ¿Los elementos de la IU tienen un nombre exacto? |
| Estilo | ¿Voz activa? ¿Presente? ¿Terminología coherente? |

### Fácil de encontrar

| Característico | Preguntas clave |
|---------------|---------------|
| Organización | ¿Estructura lógica? ¿Divulgación progresiva? ¿El más reciente primero donde corresponde? |
| Recuperabilidad | ¿Puede un lector escanear encabezados y comprender el contenido? ¿Se proporcionan vínculos? |
| Eficacia visual | ¿Listas para más de 3 elementos? ¿Tablas para comparaciones? ¿No hay párrafos de pared de texto? |

## Correcciones comunes

| Problema | Corregir |
|-------|-----|
| &quot;Estamos encantados de presentar...&quot; | Eliminar el idioma de marketing: establezca la función directamente |
| &quot;Se ha actualizado la configuración de X&quot; | &quot;Ahora puede configurar X&quot; (activo, orientado a tareas) |
| &quot;Se recomienda que...&quot; | &quot;Recomendamos...&quot; |
| &quot;Los usuarios pueden utilizar la función para...&quot; | &quot;Puede utilizar la función para...&quot; |
| &quot;establecer el valor&quot; / &quot;especificar la opción&quot; | &quot;escriba el valor&quot; / &quot;seleccione la opción&quot; |
| &quot;deshabilitado&quot; / &quot;atenuado&quot; | &quot;no disponible&quot; |
| &quot;desplegable&quot; (independiente) | &quot;menú&quot; o &quot;menú desplegable&quot; |
| &quot;él/ella&quot; | &quot;ellos&quot; o reestructurar |
| Notas apiladas | Combinar en una nota con viñetas |
| Pasos en negrita del procedimiento externo | Quitar negrita |
