# Lista de modelos disponibles

Lo siguiente [!DNL Adobe Workfront] los modelos están disponibles para que los administradores del sistema los instalen. Para obtener más información sobre los modelos, consulte [Información general sobre modelos](/help/quicksilver/administration-and-setup/blueprints/blueprints-overview.md).

**[!UICONTROL Agregar nueva lista de comprobación de TI para empleados]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de implementación con equipos de TI corporativos para organizar nuevas actividades de incorporación de empleados. El uso de esta plantilla permite a los equipos de TI operar de manera eficiente, lo que permite obtener una experiencia positiva para los empleados y un seguimiento más rápido de la productividad.

**[!UICONTROL Planificación presupuestaria anual]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de ejecución con equipos financieros para organizar las actividades relacionadas con la planificación presupuestaria anual. La plantilla ayuda a establecer expectativas con los líderes de las unidades de negocio y los guía a través del proceso para solicitar y reservar presupuestos durante el proceso de planificación anual.

**[!UICONTROL Plan de entrega de servicios contratados]**
Este modelo contiene una plantilla que organiza el trabajo relacionado con la prestación de servicios orientados al cliente. Está diseñado para crear un enfoque uniforme para la prestación de servicios contratados.

**[!UICONTROL Plan de eventos corporativos]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de ejecución con los equipos de recursos humanos para apoyar las actividades de retención de empleados, en particular los eventos corporativos y las partes.

**[!UICONTROL Desactivar lista de comprobación de TI de empleados terminados]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de implementación con equipos de TI corporativos para administrar las diversas actividades que deben realizarse cuando un empleado termina su empleo. Es fundamental desactivar el acceso de los empleados y realizar otros cambios en los sistemas para reducir la exposición al riesgo y garantizar que otros integrantes del equipo puedan seguir siendo productivos en ausencia del empleado saliente.

**[!UICONTROL Producción de recursos digitales] (Integrado)**
La variable [!UICONTROL Producción de recursos digitales] modelo para procesos integrados contiene una plantilla que organiza el trabajo para un proceso de producción digital estándar. La plantilla proporciona un proceso delgado para la creación de recursos digitales originales para un equipo que depende de procedimientos operativos estándar para definir cómo realizar el trabajo y utiliza el proyecto solo para priorizar el trabajo y rastrear el progreso.

**[!UICONTROL Producción de recursos digitales] (Administrado)**
La variable [!UICONTROL Producción de recursos digitales] contiene una plantilla que organiza el trabajo para un proceso de producción digital estándar. Se ha diseñado como punto de entrada para administrar la creación de recursos digitales en Workfront. Utilice este paquete para crear una base para la creación de contenido digital y luego personalizarlo para satisfacer las necesidades de otros procesos creativos para materiales digitales.

**[!UICONTROL Revisión de recursos digitales] (Integrado)**
La variable [!UICONTROL Revisión de recursos digitales] modelo para un proceso integrado contiene una sola plantilla para organizar el proceso de trabajo asociado con la revisión de un recurso existente. La plantilla proporciona un proceso delgado para un equipo que se basa en procedimientos operativos estándar para definir cómo realizar el trabajo y utiliza el proyecto solo para priorizar el trabajo y rastrear el progreso.

**[!UICONTROL Revisión de recursos digitales] (Administrado)**
La variable [!UICONTROL Revisión de recursos digitales] contiene una sola plantilla para organizar el trabajo asociado con una revisión de un recurso existente. La plantilla proporciona un proceso simplificado para una revisión de recursos a la vez que crea una experiencia coherente en comparación con un nuevo proyecto de producción de recursos.

**[!UICONTROL Editar lista de comprobación de TI de empleado de transferencia]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de implementación con equipos de TI corporativos para administrar las diversas actividades que deben realizarse cuando un empleado transfiere una función nueva. Esta plantilla reduce los riesgos de seguridad que existen cuando se completa una transferencia de empleados sin supervisión de TI. También elimina las horas desperdiciadas de tiempo improductivo cuando no se concede acceso a los sistemas para cumplir con las obligaciones del nuevo rol del empleado.

**[!UICONTROL Cierre operativo de fin de mes]**
Este modelo contiene una plantilla para [!DNL Workfront] la organización de servicios profesionales ha utilizado durante años los compromisos de implementación con equipos financieros para organizar el cierre de los libros a finales de mes. La plantilla proporciona una lista de verificación de las actividades comunes que deben completarse cada mes para informar y promover con precisión la salud de la empresa.

**[!UICONTROL Iniciativa estratégica interna general]**

**[!UICONTROL Imprimir: Creación de contenido de correo postal y ejecución de campañas]**
Esta [!UICONTROL Imprimir: Creación y ejecución de contenido de correo postal] modelo organiza el trabajo para un proceso de producción de material publicitario impreso. Este modelo está diseñado como punto de entrada para administrar material impreso, específicamente correo postal, en Workfront. Esta plantilla es única porque también contiene tareas para imprimir el recurso y ejecutar la campaña de correo postal independientemente de otras actividades de campaña.

**[!UICONTROL Autoevaluación del control de riesgos: Planificación de la reducción de riesgos clave]**
[!UICONTROL Autoevaluación del control de riesgos] (RSCA) define un proceso para identificar los riesgos y medir los controles que se aplican para mitigar dichos riesgos. Esta plantilla formaliza el proceso para evaluar nuevos riesgos, para garantizar que se establecen los controles adecuados y para asegurarse de que sabe cómo medir y documentar la eficacia del control.

**[!UICONTROL Autoevaluación del control de riesgos: Acreditación trimestral]**
[!UICONTROL Autoevaluación del control de riesgos] (RSCA) define un proceso para identificar los riesgos y medir los controles que su organización tiene establecidos para mitigar esos riesgos. Esta plantilla proporciona una estructura que le ayuda a crear un registro de riesgo inicial, a establecer un plan para mitigar cada riesgo y a proporcionar documentación para mostrar el trabajo de los controles.
