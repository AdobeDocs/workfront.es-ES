# Información general sobre las fechas del proyecto, la tarea y el problema en [!DNL Adobe Workfront]

<!--
<p data-mc-conditions="QuicksilverOrClassic.Draft mode">(NOTE: consider expanding on this article with ALL dates for PTIs - Hand off dates, Approval Dates, etc) </p>
-->

Este artículo proporciona definiciones de las fechas más comunes asociadas con proyectos, tareas y problemas en [!DNL Adobe Workfront]. Las imágenes incluidas aquí son ejemplos de dónde se muestran las fechas en Workfront y no son exhaustivas. Hay otras áreas que muestran las fechas. Todas las fechas también están visibles en los informes y listas de proyectos, tareas y problemas.

Para obtener información sobre informes y listas, consulte los siguientes artículos:

* [Introducción a las listas en [!DNL Adobe Workfront]](../../../workfront-basics/navigate-workfront/use-lists/view-items-in-a-list.md)
* [Introducción a los informes en [!DNL Adobe Workfront]](../../../reports-and-dashboards/reports/reporting/get-started-reports-workfront.md)

Para obtener más información sobre los campos de proyecto, tarea y problema, consulte [Glosario de [!DNL Adobe Workfront] terminología](../../../workfront-basics/navigate-workfront/workfront-navigation/workfront-terminology-glossary.md).

## [!UICONTROL Fecha planificada de inicio]

La variable [!UICONTROL Fecha de inicio planeada] es la fecha en la que se planea iniciar un proyecto, una tarea o un problema.

Según el [!UICONTROL Restricción de tarea], es posible que no pueda editar la variable [!UICONTROL Fecha de inicio planeada] de una tarea. Según el [!UICONTROL Modo de programación] del proyecto, es posible que no pueda editar la variable [!UICONTROL Fecha de inicio planeada] de un proyecto.

Para obtener más información, consulte [Descripción general del proyecto [!UICONTROL Fecha de inicio planeada]](../../../manage-work/projects/planning-a-project/project-planned-start-date.md).

![](assets/planned-start-date-on-edit-task-highlighted-nwe-350x249.png)

![](assets/planned-start-date-in-task-list-highlighted-nwe-350x167.png)

## [!UICONTROL Fecha planificada de finalización]

La variable [!UICONTROL Fecha de finalización planeada] o [!UICONTROL Vence el] fecha es la fecha en la que se planea completar un proyecto, una tarea o un problema.

Según el [!UICONTROL Restricción de tarea], es posible que no pueda editar la variable [!UICONTROL Fecha de finalización planeada] de una tarea. Según el [!UICONTROL Modo de programación] del proyecto, es posible que no pueda editar la variable [!UICONTROL Fecha de finalización planeada] de un proyecto.

La variable [!UICONTROL Fecha de finalización planeada] se muestra como la fecha de vencimiento en algunas áreas de [!DNL Workfront].

Para obtener más información, consulte los siguientes artículos:

* [Descripción general de la tarea [!UICONTROL Fecha de finalización planeada]](../../../manage-work/tasks/task-information/task-planned-completion-date.md)
* [Definir el proyecto [!UICONTROL Fecha de finalización planeada]](../../../manage-work/projects/planning-a-project/project-planned-completion-date.md)
* [Descripción general del problema [!UICONTROL Fecha de finalización planeada]](../../../manage-work/issues/issue-information/issue-planned-completion-date.md)

![](assets/project-header-planned-completion-date-highlighted-nwe-350x34.png)

![](assets/planned-completion-date-in-task-list-highlighted-nwe-350x183.png)

## [!UICONTROL Fecha de entrada]

La variable [!UICONTROL Fecha de entrada] es la fecha en la que se creó un proyecto, una tarea o un problema en Workfront.

La variable [!UICONTROL Fecha de entrada] no influye en la cronología de los proyectos, tareas o problemas, pero es importante para fines de seguimiento e informes. [!DNL Workfront] genera automáticamente la variable [!UICONTROL Fecha de entrada] cuando se crea el objeto y no se puede editar manualmente.

![](assets/entry-date-in-task-details-highlighted-nwe-350x105.png)

## [!UICONTROL Fecha real de inicio]

La variable [!UICONTROL Fecha de inicio real] es la fecha en la que un usuario comienza a trabajar en un proyecto, una tarea o un problema. La variable [!UICONTROL Fecha de inicio real] está vacío cuando se crea el proyecto, la tarea o el problema.

Puede indicar manualmente cuándo se inició el trabajo en una tarea o un problema, o cuándo [!UICONTROL Fecha de inicio real] se rellena automáticamente cuando cambia el estado de la tarea o del problema [!UICONTROL Nuevo] a [!UICONTROL En curso] o [!UICONTROL Completar]. La variable [!UICONTROL Fecha de inicio real] de un proyecto coincide con la fecha en que se inicia la primera tarea del proyecto.

>[!TIP]
>
>La variable [!UICONTROL Fecha de inicio real] puede no coincidir con [!UICONTROL Fecha de inicio planeada] de un proyecto, tarea o problema porque el usuario puede iniciar el trabajo más tarde o antes de la fecha prevista.

Para obtener más información, consulte [Descripción general del proyecto [!UICONTROL Fecha de inicio real]](../../../manage-work/projects/planning-a-project/project-actual-start-date.md).

>[!NOTE]
>
>La variable [!UICONTROL Debe comenzar el] o las restricciones de Fechas fijas afectan a la función [!UICONTROL Fecha de inicio planeada] de una tarea, no el [!UICONTROL Fecha de inicio real]. Esto actualiza el [!UICONTROL Fecha de inicio planeada] a la fecha especificada. La variable [!UICONTROL Fecha de inicio real] se actualiza independientemente del [!UICONTROL Fecha de inicio planeada], tal como se ha descrito anteriormente.

![](assets/actual-start-date-on-edit-task-highlighted-nwe-350x251.png)

![](assets/actual-start-date-on-task-details-highlighted-nwe-350x191.png)

## [!UICONTROL Fecha real de finalización]

La variable [!UICONTROL Fecha de finalización real] es la fecha en la que un usuario completa un proyecto, una tarea o un problema. La variable [!UICONTROL Fecha de finalización real] está vacío cuando se crea el proyecto, la tarea o el problema.

Puede indicar manualmente cuándo se completa el trabajo en una tarea o un problema, o cuándo [!UICONTROL Fecha de finalización real] se rellena automáticamente cuando se produce cualquiera de las siguientes situaciones:

* El estado del proyecto, la tarea o el problema cambia a [!UICONTROL Completar], [!UICONTROL Cerrado]o [!UICONTROL Resuelto].
* El porcentaje completado de la tarea o del proyecto es del 100%.

La variable [!UICONTROL Fecha de finalización real] de un proyecto coincide con la fecha en que se completa la primera tarea del proyecto.

>[!TIP]
>
>La variable [!UICONTROL Fecha de finalización real] puede que no coincida con la variable [!UICONTROL Fecha de finalización planeada].

Para obtener más información, consulte [Descripción general del proyecto [!UICONTROL Fecha de finalización real]](../../../manage-work/projects/planning-a-project/project-actual-completion-date.md).

![](assets/actual-completion-date-task-details-highlighted-nwe-350x189.png)

## [!UICONTROL Fecha de confirmación]

La variable [!UICONTROL Fecha de confirmación] es la fecha en la que un usuario asignado a una tarea o un problema se compromete a completar la tarea o el problema. Es diferente a la función [!UICONTROL Fecha de finalización planeada], ya que es una estimación más realista de la fecha de finalización dada únicamente por el usuario a cargo del trabajo. Para obtener más información, consulte [[!UICONTROL Fecha de confirmación] información general](../../../manage-work/projects/updating-work-in-a-project/overview-of-commit-dates.md).

>[!NOTE]
>
>Cambio de la variable [!UICONTROL Fecha de confirmación] afecta a la variable [!UICONTROL Fecha de finalización prevista] pero no la variable [!UICONTROL Fecha de finalización planeada] de una tarea o un problema. El administrador de proyectos puede utilizar los cambios que realiza un usuario asignado en la variable [!UICONTROL Fecha de confirmación] para actualizar el [!UICONTROL Fecha de finalización planeada] de una tarea o un problema.

![](assets/commit-date-on-task-details-highlighted-nwe-350x193.png)

![](assets/commit-date-in-updates-stream-on-task-nwe-350x344.png)

## [!UICONTROL Fecha proyectada de inicio]

La variable [!UICONTROL Fecha inicial prevista] es una fecha en tiempo real en la que el proyecto, la tarea o el problema se producen y tiene en cuenta todos los retrasos. Esta es una fecha de inicio más precisa para el proyecto, la tarea o el problema que la del [!UICONTROL Fecha de inicio planeada]. La variable [!UICONTROL Fecha de inicio planeada] no tiene en cuenta los retrasos o las fechas pasadas.

Cuando planifique por primera vez un proyecto, la variable [!UICONTROL Fecha de inicio planeada] y [!UICONTROL Fecha inicial prevista] de las tareas y del proyecto son idénticos. Como pueden producirse retrasos o las tareas podrían completarse antes, la variable [!UICONTROL Fecha inicial prevista] puede ser diferente al [!UICONTROL Fecha de inicio planeada].

Para una tarea, una [!UICONTROL Fecha inicial prevista] también pueden diferir de su [!UICONTROL Fecha de inicio planeada] cuando uno de sus predecesores está retrasado.

>[!TIP]
>
>Puede ver la [!UICONTROL Fecha inicial prevista] de un problema solo en una lista o informe.

Para obtener más información, consulte [Descripción general del proyecto [!UICONTROL Fecha inicial prevista]](../../../manage-work/projects/planning-a-project/project-projected-start-date.md).

![](assets/projected-start-date-in-task-details-highlighted-nwe-350x188.png)

## P[!UICONTROL Fecha de finalización prevista]

La variable [!UICONTROL Fecha de finalización prevista] es un indicador calculado en tiempo real de cuándo se completará el proyecto, la tarea o el problema. Cuando el proyecto, la tarea o el problema se marcan como Completado, la variable [!UICONTROL Fecha de finalización prevista] cambios en la fecha de [!UICONTROL Fecha de finalización real].

Si todo va sin problemas y como está previsto, la variable [!UICONTROL Fecha de finalización prevista] debe coincidir con la variable [!UICONTROL Fecha de finalización planeada]. De lo contrario, debido a los retrasos en las tareas predecesoras, la función [!UICONTROL Fecha de finalización prevista] puede diferir de la función [!UICONTROL Fecha de finalización planeada].

Para obtener más información, consulte [Información general sobre [!UICONTROL Fecha de finalización prevista] para proyectos, tareas y problemas](../../../manage-work/projects/planning-a-project/project-projected-completion-date.md).

![](assets/projected-completion-date-in-task-details-highlighted-nwe-350x187.png)

## [!UICONTROL Fecha de entrada de hora]

Cuando se registra el tiempo para proyectos, tareas y problemas para indicar cuánto tiempo real (en horas) se pasa trabajando en el proyecto, la tarea o el problema, el tiempo que se registra se convierte en el valor [!UICONTROL Horas reales] del proyecto, tarea o problema.

La fecha en la que se registra la hora es la [!UICONTROL Fecha de entrada de hora] en la entrada de hora.

>[!TIP]
>
>La variable [!UICONTROL Fecha de entrada de hora] es diferente a [!UICONTROL Fecha de entrada] en el sentido de que no es la fecha en la que se creó el registro de horas, sino la fecha a la que desea asociar las horas.

Puede iniciar sesión en la [!UICONTROL proyecto], [!UICONTROL tarea]o [!UICONTROL actualizaciones de problemas] o en la sección [!UICONTROL Horas] sección . Al iniciar sesión, la variable [!UICONTROL Horas] , puede especificar manualmente la fecha de entrada de la hora y el usuario al que pertenecen las horas.

Para obtener más información, consulte [Tiempo de registro](../../../timesheets/create-and-manage-timesheets/log-time.md).

>[!TIP]
>
>Se recomienda iniciar sesión en tareas y problemas de trabajo, en lugar de en tareas o proyectos principales. El tiempo registrado en las tareas de trabajo se resume en las tareas principales y el proyecto como [!UICONTROL Horas reales] para las tareas principales y el proyecto. El tiempo que se ha iniciado sesión en los problemas se resume en el proyecto como [!UICONTROL Horas reales] para el proyecto.

![](assets/log-time-box-task-hours-section-nwe-350x500.png)

![](assets/log-time-in-update-stream-task-nwe-350x185.png)

Puede ver la [!UICONTROL Fechas de entrada por hora] en listas e informes de hora.
