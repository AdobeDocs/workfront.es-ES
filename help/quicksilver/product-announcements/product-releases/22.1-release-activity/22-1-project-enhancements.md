# 22.1 Mejoras del proyecto

En esta página se describen todas las mejoras del proyecto realizadas con la versión 2.1 del entorno de vista previa. Estas mejoras estarán disponibles en el entorno Producción

in January 2022
</MadCap:conditionalText>

la semana del 17 de enero de 2022.

Para obtener una lista de todos los cambios disponibles con la versión 2.1, consulte [Resumen de la versión 2.1](../../../product-announcements/product-releases/22.1-release-activity/22-1-release-overview.md).

## Cambiar la opción predeterminada para predecesores al copiar o mover tareas

Para minimizar el número de pasos manuales al copiar o mover tareas, hemos actualizado la información que se copia o mueve con la tarea de forma predeterminada. Ahora, todos los predecesores se copian o se mueven con la tarea de forma predeterminada, ya que la opción Todos los predecesores está seleccionada de forma predeterminada.

Antes de esta mejora, la opción All Predecesors no estaba seleccionada de forma predeterminada al copiar o mover una tarea.

Para obtener más información, consulte los artículos:

* [Copiar y duplicar tareas](../../../manage-work/tasks/manage-tasks/copy-and-duplicate-tasks.md)
* [Mover tareas](../../../manage-work/tasks/manage-tasks/move-tasks.md)

## Se ha actualizado la barra de herramientas de la lista de tableros y los informes de los tableros

La barra de herramientas de cuatro páginas del tablero tiene ahora un aspecto moderno que coincide con otras listas de Workfront, como proyectos, tareas y problemas. Esta intuitiva barra de herramientas ahora facilita la adición, edición, uso compartido, copia y eliminación de tableros.

Las páginas con la barra de herramientas actualizada son:

* Informes de tareas (mostrados en tableros)
* Lista de todos los tableros
* Lista Mis tableros
* Lista de tableros compartidos

Para obtener más información, consulte [Crear y administrar tableros](../../../reports-and-dashboards/dashboards/creating-and-managing-dashboards/create-and-manage-dashboards.md).

## Agregar y editar formularios personalizados Panel de resumen para documentos

Ahora puede agregar y editar formularios personalizados directamente en el panel Resumen del documento.

Con este cambio, también verá un aspecto nuevo en el resumen del documento. Hay una nueva sección Información general que contiene la miniatura de la imagen y los detalles del documento. También puede registrar los documentos dentro y fuera de la sección de detalles del documento.

Anteriormente, tenía que ir a la pestaña formularios personalizados de Detalles del documento para realizar ediciones o agregar formularios personalizados.

Para obtener más información, consulte [Resumen de los documentos](../../../documents/managing-documents/summary-for-documents.md).

## Nueva experiencia al copiar una o varias tareas

Para que el uso de Workfront sea coherente con la nueva experiencia de Adobe Workfront, hemos rediseñado la interfaz al copiar una tarea. Actualmente está disponible al copiar una tarea en el nivel de tarea o al copiar una tarea o varias tareas en una lista.

Algunas de las mejoras incluyen:

* Toda la información que debe actualizar antes de copiar la tarea se muestra en una página continua.
* Workfront comprueba si tiene acceso al proyecto de destino inmediatamente después de elegir el proyecto. Antes de esta mejora, aparece un mensaje de advertencia que indica que no tiene el acceso correcto después de confirmar la copia, lo que da como resultado pasos adicionales y que la copia no está permitida.
* Capacidad para solicitar acceso a un proyecto en el que desee copiar la tarea sin salir del cuadro Copiar tarea.

Para obtener más información, consulte [Copiar y duplicar tareas](../../../manage-work/tasks/manage-tasks/copy-and-duplicate-tasks.md).

## Nueva experiencia al mover una o varias tareas de una lista

Para ofrecer una experiencia coherente al realizar la misma tarea, ahora hemos actualizado la interfaz para mover una o varias tareas de una lista para que coincidan con la experiencia al mover la tarea al nivel de tarea. (Hemos actualizado la experiencia para mover una tarea a nivel de tarea en una versión anterior de Vista previa).

Para obtener más información, consulte [Mover tareas](../../../manage-work/tasks/manage-tasks/move-tasks.md).

## Nueva experiencia al mover una tarea al nivel de tarea

Para que el uso de Workfront sea coherente con la nueva experiencia de Workfront, hemos rediseñado la interfaz para mover una tarea. Actualmente está disponible al mover una tarea al nivel de tarea. En una versión siguiente, ampliaremos este rediseño para mover una tarea de una lista.

Algunas de las mejoras de esta interfaz recién rediseñada incluyen:

* Toda la información que debe actualizar antes del movimiento se muestra en una página continua.
* Workfront comprobará si tiene acceso al proyecto de destino inmediatamente después de elegir el proyecto. Antes de esta mejora, Workfront le advirtió de que no tiene el acceso correcto después de confirmar el movimiento, lo que provocaba que se realizaran pasos adicionales y que el movimiento no estuviera permitido.
* Capacidad para solicitar acceso a un proyecto en el que desee mover la tarea sin salir del cuadro Mover tarea .

Para obtener más información, consulte [Mover tareas](../../../manage-work/tasks/manage-tasks/move-tasks.md).

## Nueva experiencia al convertir un problema en un proyecto mediante una plantilla a nivel de problema

>[!NOTE]
>
>Esta función se eliminó temporalmente del entorno Producción el 4 de marzo de 2022. Posteriormente se publicó en una implementación gradual a partir del 28 de abril de 2022. El lanzamiento finalizó el 5 de mayo de 2022. Ahora está disponible en Vista previa y Producción para todos los clientes. (Para obtener las últimas actualizaciones sobre el estado de la versión de esta función a Producción, consulte [Información general sobre la versión 22.3](../../../product-announcements/product-releases/22.3-release-activity/22-3-release-overview.md).)

Para que el uso de Workfront sea coherente con la nueva experiencia de Workfront, hemos rediseñado la interfaz para convertir un problema en un proyecto al usar una plantilla cuando la convierte desde la página de problemas.

Ahora puede acceder más fácilmente a su lista de favoritos inmediatamente después de seleccionar para convertir el problema.

La interfaz rediseñada coincide con la experiencia al crear un proyecto a partir de una plantilla que también hemos actualizado recientemente.

Para obtener más información, consulte [Convertir un problema en un proyecto en Adobe Workfront](../../../manage-work/issues/convert-issues/convert-issue-to-project.md).

## Convertir problemas en proyectos mediante una plantilla de listas, informes y tableros

>[!NOTE]
>
>Esta función se eliminó temporalmente del entorno Producción el 4 de marzo de 2022. Posteriormente se publicó en una implementación gradual a partir del 28 de abril de 2022. El lanzamiento finalizó el 5 de mayo de 2022. Ahora está disponible en Vista previa y Producción para todos los clientes. (Para obtener las últimas actualizaciones sobre el estado de la versión de esta función a Producción, consulte [Información general sobre la versión 22.3](../../../product-announcements/product-releases/22.3-release-activity/22-3-release-overview.md).)

Para aumentar la eficacia de su trabajo y facilitar la conversión de problemas en un entorno de ritmo acelerado, se ha añadido la capacidad de convertir un problema en un proyecto mediante una plantilla de una lista, un informe o un panel.

Antes de esta mejora, esta funcionalidad solo existía cuando se convertía el problema desde la página de problemas.

Para obtener más información, consulte [Convertir un problema en un proyecto en Adobe Workfront](../../../manage-work/issues/convert-issues/convert-issue-to-project.md).

## Filtrar por lista de usuarios en tableros ágiles muestra primero a los usuarios con la mayoría de asignaciones

>[!NOTE]
>
>Esta función no se incluirá en la versión 2.1. Se eliminó del entorno de vista previa.

Ahora el filtro muestra primero a los usuarios con la mayor cantidad de asignaciones para que sean más fáciles de localizar sin desplazarse por la lista.

Anteriormente, el filtro por lista de usuarios en los tableros Kanban y Scrum se mostraba alfabéticamente.

Para obtener más información, consulte la siguiente información

* [Filtrar por usuario en el panel Anular](../../../agile/use-scrum-in-an-agile-team/scrum-board/filter-by-user-scrum-board.md)
* [Filtrar por usuario en el panel Kanban](../../../agile/use-kanban-in-an-agile-team/filter-by-user.md)

## Limitar la capacidad de agregar documentos a una plantilla que esté compartiendo

A veces, las personas agregan documentos a una plantilla de proyecto pensando que los están agregando a un proyecto. Ahora puede ayudar a evitar esto: cuando comparte una plantilla con acceso de vista, puede desactivar la nueva configuración avanzada Agregar documentos. Esto deshabilita la capacidad de los destinatarios para agregar documentos a la plantilla.

Para obtener instrucciones sobre cómo compartir una plantilla, consulte [Compartir plantillas de proyecto](../../../manage-work/projects/create-and-manage-templates/share-project-template.md).

Para obtener información sobre la nueva configuración avanzada Agregar documentos, consulte la sección del artículo [Uso compartido de una plantilla](../../../workfront-basics/grant-and-request-access-to-objects/share-a-template.md).

## Compartir una carpeta de documento

Ahora puede compartir una carpeta de documento y su contenido desde el área Documentos . Anteriormente, esto no era posible: tenía que compartir cada documento en una carpeta por separado.

Para obtener más información, consulte [Compartir una carpeta de documento](../../../workfront-basics/grant-and-request-access-to-objects/share-a-document-folder.md).
