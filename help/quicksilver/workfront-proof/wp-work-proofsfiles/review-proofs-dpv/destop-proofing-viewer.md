# Información sobre el visor de corrección de escritorio

>[!IMPORTANT]
>
>Este artículo hace referencia a la funcionalidad del producto independiente [!DNL Workfront Proof]. Para obtener información sobre la revisión dentro de [!DNL Adobe Workfront], consulte [Revisión](../../../review-and-approve-work/proofing/proofing.md).

El Visor de corrección de escritorio está diseñado para revisar el contenido interactivo, pero también puede utilizarlo para revisar contenido estático y de vídeo.

A diferencia del Visor de revisión web, el Visor de revisión de escritorio es una aplicación que se ejecuta en la estación de trabajo. Sin embargo, accede a él desde [!DNL Workfront] o desde [!DNL Workfront Proof] cuando inicia las pruebas, en lugar de ejecutarlo como una aplicación independiente (para obtener más información sobre la instalación del Visor de pruebas de escritorio, consulte [Instalar el Visor de pruebas de escritorio](../../../review-and-approve-work/proofing/use-the-desktop-proofing-viewer/installing-desktop-proofing-viewer.md)).

## Comparación del Visor de corrección de escritorio con otros visores de corrección

* [Visor de corrección de escritorio frente a Visor de corrección de Web](#desktop-proofing-viewer-vs-web-proofing-viewer)
* [Visor de corrección de escritorio frente a visor de revisión heredado](#desktop-proofing-viewer-vs-legacy-proofing-viewer)

### Visor de corrección de escritorio frente a Visor de corrección de Web {#desktop-proofing-viewer-vs-web-proofing-viewer}

Dos características principales distinguen al Visor de corrección de escritorio del Visor de corrección web:

* El Visor de corrección de escritorio le permite revisar contenido interactivo, como un sitio web, un vídeo de flujo continuo o un banner interactivo; el Visor de corrección web le permite revisar únicamente pruebas estáticas y de vídeo.
* El Visor de corrección de escritorio se ejecuta en el equipo local como una aplicación; el Visor de corrección web se ejecuta en el explorador.

  >[!NOTE]
  >
  >   * Si su organización no puede usar la aplicación Visor de corrección de escritorio por motivos de seguridad, su administrador de [!DNL Workfront] podrá configurar su sistema para que pueda revisar el contenido interactivo, incluido en un archivo ZIP, en el Visor de corrección de Web.
  >   * Las tareas de administración de pruebas, como crear y examinar pruebas, se realizan en [!DNL Workfront] o [!DNL Workfront Proof]. El Visor de corrección de escritorio solo sirve para revisar pruebas.


Para obtener más información acerca de las capacidades de los distintos visores de revisión, vea [Diferencias entre el Visor de revisión web y el Visor de revisión de escritorio](../../../review-and-approve-work/proofing/proofing-overview/understand-differences-between-web-viewer.md)

### Visor de corrección de escritorio frente a visor de revisión heredado {#desktop-proofing-viewer-vs-legacy-proofing-viewer}

Es posible que no tenga acceso al visor de revisión heredado porque es compatible con Flash, que ha quedado obsoleto en la mayoría de los entornos.

El Visor de corrección de escritorio incluye las siguientes mejoras con respecto al Visor de corrección de elementos heredados:

* Revise los sitios no seguros (HTTP), no solo los sitios seguros (HTTPS).
* Revisar sitios protegidos por iFrame (sitios que no se pueden ver dentro de un iFrame).
* Ver contenido con resoluciones preconfiguradas para varios dispositivos.\
   Por ejemplo, puede ver cómo se muestra el contenido en varias resoluciones de escritorio estándar o en dispositivos individuales como un iPhone 8.

El visor de revisión heredado se eliminará de [!DNL Workfront] con la versión 2018.3 al final del año natural 2018.

El visor de revisión heredado no se incluye para los clientes que adquieren [!DNL Workfront] después de la versión 2018.2 en julio de 2018.

## Configuración de usuario para abrir pruebas no interactivas en el visualizador de pruebas de escritorio

Aunque el propósito principal del Visor de corrección de escritorio es la corrección del contenido interactivo, también puede utilizarlo para revisar pruebas estáticas y de vídeo.

Los usuarios pueden configurar el Visor de pruebas de escritorio para que se inicie automáticamente cuando abran pruebas de todo tipo, incluidas las pruebas estáticas y de vídeo. Para obtener más información, consulte [Configuración de ajustes del visualizador de revisión](../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/configure-proofing-viewer-settings.md).

## Introducción de una URL de prueba

Si inicia el Visor de pruebas de escritorio directamente desde la estación de trabajo, puede abrir una prueba especificando la dirección URL de prueba. Para ello, debe tener la dirección URL de prueba de la prueba que desea ver.

Le recomendamos que abra pruebas directamente desde [!DNL Workfront]. Para obtener más información, vea [Revisar una revisión](../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/review-a-proof.md).
