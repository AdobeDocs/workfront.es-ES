# Preguntas frecuentes - Crear y compartir pruebas y archivos

>[!IMPORTANT]
>
>Este artículo hace referencia a la funcionalidad del producto independiente [!DNL Workfront Proof]. Para obtener información sobre la revisión dentro de [!DNL Adobe Workfront], vea [Revisión](../../../review-and-approve-work/proofing/proofing.md).

## ¿Qué es una prueba?

### Respuesta

Una prueba es un archivo estático, de audio o de vídeo disponible para su revisión en el visor de pruebas. Los revisores agregados a una prueba tienen disponible un conjunto de herramientas para realizar comentarios y tomar decisiones al respecto.

## ¿Qué tipos de archivo se admiten?

### Respuesta

Se pueden crear pruebas a partir de archivos estáticos, de audio y de vídeo. No puede cargar archivos de más de 4 GB. [!DNL Workfront] admite más de 150 tipos de archivo (consulte [Información general sobre los tipos de archivo de revisión admitidos y los límites de tamaño](../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md) para obtener una lista completa).

## ¿Cuál es la diferencia entre una prueba y un archivo?

### Respuesta

Al cargar un archivo en [!DNL Workfront Proof], el sistema lo almacena en su cuenta de [!DNL Workfront Proof]. Al compartirlo, [!DNL Workfront Proof] envía un mensaje de correo electrónico a los destinatarios con un vínculo en el que pueden hacer clic para descargar el archivo. Puede compartir cualquier tipo de archivo que desee.

Cuando cree una prueba a partir de un archivo que haya cargado en [!DNL Workfront Proof], puede hacer que el archivo esté disponible para su revisión en el visor de pruebas. Los revisores reciben un correo electrónico con un vínculo a la prueba. Cuando abren la prueba, ven la imagen de la prueba y pueden agregar comentarios y tomar decisiones al respecto. Puede crear pruebas utilizando archivos de la lista de tipos de archivo admitidos. También puede crear pruebas utilizando direcciones URL para sitios web y otro contenido web.

Para obtener una lista completa de los tipos de archivo admitidos, consulte [Información general sobre los tipos de archivo de revisión admitidos y los límites de tamaño](../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md).

## ¿Cómo se crea una prueba?

### Respuesta

Puede crear pruebas a partir de archivos estáticos, archivos de audio, archivos de vídeo y direcciones URL (consulte ).

Para crear una revisión en la cuenta, debes ser un usuario con el perfil de permiso correcto (consulta [[!UICONTROL Perfiles de permisos de revisión] en [!DNL Workfront Proof]](../../../workfront-proof/wp-acct-admin/account-settings/proof-perm-profiles-in-wp.md)).

Si carga varios archivos al mismo tiempo, crea varias pruebas que puede enviar al mismo grupo de revisores con un solo correo electrónico. Si su organización tiene una cuenta de [!UICONTROL Enterprise] o [!UICONTROL Unlimited], puede combinar archivos en una sola revisión (consulte [Crear una revisión de varias páginas](../../../review-and-approve-work/proofing/creating-proofs-within-workfront/create-multi-page-proof.md)).

## ¿Qué son las funciones de prueba y las alertas por correo electrónico?

### Respuesta

Las funciones de prueba definen qué acciones debe realizar un revisor en una prueba. Existen diferentes opciones de función que puede utilizar para los revisores cuando cree una prueba, en función de si desea que puedan realizar comentarios, tomar decisiones, etc. Para obtener más información, consulte [Administrar roles de prueba en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/manage-proof-roles.md).

Las alertas de correo electrónico actualizan a los revisores sobre el progreso de una prueba (son diferentes de las notificaciones de nuevas pruebas y de pruebas tardías). Puede seleccionar diferentes opciones para diferentes revisores, según la función de cada revisor en la prueba. Para obtener más información, consulte [Crear una revisión avanzada con un [!UICONTROL flujo de trabajo automatizado]](../../../review-and-approve-work/proofing/creating-proofs-within-workfront/create-automated-proof-workflow.md).

## ¿Puedo crear una prueba a partir de varios archivos?

### Respuesta

Combinar varios archivos en una sola revisión es una característica disponible en los planes de edición [!UICONTROL Enterprise] y [!UICONTROL Unlimited]. Esta opción solo es posible para archivos estáticos, no para archivos de vídeo. Para obtener más información, consulte [Crear una revisión de varias páginas](../../../review-and-approve-work/proofing/creating-proofs-within-workfront/create-multi-page-proof.md).

## ¿Puedo crear pruebas a partir de direcciones URL?

### Respuesta

Sí, puede crear pruebas a partir de sitios web y otro contenido web. Cuando añada una dirección URL para crear una prueba, puede especificar si desea una prueba estática o interactiva:

* En una revisión interactiva, los revisores pueden navegar e interactuar como lo harían normalmente con el sitio web u otro contenido web, como anuncios con flujo de vídeo o audio, [!DNL Flash] elementos en un anuncio, animaciones de HTML y titulares interactivos. Para obtener más información, consulte [Crear una prueba para contenido interactivo en un archivo ZIP](../../../review-and-approve-work/proofing/creating-proofs-within-workfront/generate-proof-interactive-content.md).
* Para una revisión estática, [!DNL Workfront] toma un conjunto de capturas de pantalla de las páginas y subpáginas que especifique. Los hipervínculos están activos en la prueba, por lo que puede probar si conducen o no al destino correcto. Para obtener más información, consulte [Crear una revisión estática para un sitio web u otro contenido web](../../../review-and-approve-work/proofing/creating-proofs-within-workfront/generate-static-proof-website-other-web-content.md).

Puede añadir varias direcciones URL a la vez si las separa con un espacio. Tenga en cuenta que la combinación solo está disponible en los planes de edición [!UICONTROL Enterprise] y [!UICONTROL Unlimited].

## ¿Pueden las personas que no tienen un inicio de sesión crear pruebas en mi cuenta?

### Respuesta

Necesita credenciales de inicio de sesión para crear pruebas directamente en una cuenta de [!DNL Workfront Proof].

## ¿Qué significa compartir una prueba?

### Respuesta

Al compartir una prueba, los revisores tienen acceso a ella para que puedan agregar comentarios y marcas y tomar decisiones al respecto. Los revisores invitados acceden a las pruebas desde la notificación por correo electrónico que reciben. Los revisores que tengan su propia cuenta de [!DNL Workfront Proof] podrán obtener acceso a las pruebas en el [!UICONTROL panel].

## ¿Cómo comparto una prueba?

### Respuesta

Cuando esté creando una prueba, puede agregar revisores en la sección [!UICONTROL Flujo de trabajo] de la página [!UICONTROL Nueva prueba]. Cuando la revisión esté lista, [!DNL Workfront Proof] enviará un correo electrónico a los revisores con un vínculo a la revisión.

Si dispone de derechos suficientes sobre una prueba, puede usar el visor de pruebas, el [!UICONTROL panel] o cualquiera de las vistas de lista para agregar revisores a una prueba existente (consulte [Compartir una prueba en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/share-proof.md) and [Manage Items on the [!UICONTROL Views] Página en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/manage-your-work/manage-items-on-views-page.md).

La forma más común de compartir pruebas es agregar revisores. Si desea explorar otras opciones disponibles, consulte:

* [Compartir vínculos de revisión en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/share-proof-links.md)
* [Compartir la dirección URL pública en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/share-public-url.md)
* [Suscribirse a una revisión en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/subscribe-to-proof.md)
* [Crear una revisión mínima en  [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/share-proofs-and-files/create-mini-proof.md)

## ¿Tiene que ser un usuario para revisar una prueba?

### Respuesta

No. Los revisores invitados (las personas sin [!DNL Workfront Proof] credenciales de inicio de sesión) pueden acceder a una revisión desde la notificación de prueba por correo electrónico que reciben. Puede compartir una prueba con tantos invitados como desee.

Es posible restringir el uso compartido de pruebas a personas con [!DNL Workfront Proof] credenciales de inicio de sesión. Esto añade otra capa de seguridad a las pruebas. Para mejorar la seguridad, los administradores de sistemas de las organizaciones con planes [!UICONTROL Enterprise] y [!UICONTROL Unlimited] pueden configurar este requisito para todas las pruebas creadas en la organización.

Para obtener más información acerca de cómo requerir el inicio de sesión, vea [Seguridad de pruebas en [!DNL Workfront Proof]](../../../workfront-proof/wp-acct-admin/managing-security/proof-security-in-workfront-proof.md).

Si su organización requiere que los revisores firmen las pruebas electrónicamente, para lo cual es necesario iniciar sesión en [!DNL Workfront Proof], los usuarios solo podrán compartir las pruebas con los usuarios registrados. Esto está disponible en los planes de edición [!UICONTROL Enterprise] y [!UICONTROL Unlimited]. Para obtener más información, vea [Explicación de las firmas electrónicas en [!DNL Workfront Proof]](../../../workfront-proof/wp-acct-admin/managing-security/electronic-sigs-in-wp.md).

## ¿Puedo establecer una fecha límite para mis revisores?

### Respuesta

Puede establecer una fecha límite para una nueva prueba o una nueva versión de prueba al crear la prueba. Esto se hace en la sección [!UICONTROL Flujo de trabajo] de la página [!UICONTROL Nueva prueba]. Si usa [!UICONTROL Flujo de trabajo automatizado], puede establecer una fecha límite diferente para cada etapa de la revisión.

También puede establecer o actualizar una fecha límite para una revisión existente mediante la página [!UICONTROL Detalles de la revisión]. Para obtener más información, consulte [Administrar detalles de revisión en [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/manage-your-work/manage-proof-details.md).

## ¿Cómo puedo crear una nueva versión de mi revisión?

### Respuesta

Los revisores suelen solicitar cambios en sus comentarios sobre una prueba y desean ver una nueva versión de la prueba. Puede crear nuevas versiones de una prueba. [!DNL Workfront Proof] recuerda la configuración de la revisión de la versión anterior. Puede seguir editando esta configuración si necesita hacer algo como agregar o quitar revisores para la prueba.

Debe compartir cada versión con los revisores específicos que necesiten verla. Por ejemplo, si comparte únicamente la versión 3 con un revisor, esa persona no puede ver las versiones 1 y 2. Los supervisores y administradores de su cuenta supervisan todos los proyectos de la cuenta para que puedan ver y editar todas las versiones de la prueba.

Para obtener más información, consulte .

## ¿Puedo compartir archivos usando [!DNL Workfront Proof]?

### Respuesta

Sí. Si desea compartir algo con otras personas, pero no necesita que lo vean como una prueba (o si es un tipo de archivo no admitido por [!DNL Workfront Proof]), puede cargarlo como un archivo en su cuenta de [!DNL Workfront Proof]. Al igual que con las pruebas, puede organizar los archivos en carpetas, etiquetar archivos y agregar un mensaje personalizado al mensaje de correo electrónico de notificación cuando comparta el archivo. Para obtener más información, consulte [Cargar archivos y contenido web a [!DNL Workfront Proof]](../../../workfront-proof/wp-work-proofsfiles/create-proofs-and-files/upload-files-web-content.md).

Cuando los destinatarios reciben la notificación por correo electrónico de un archivo que está compartiendo, pueden descargarlo haciendo clic en el vínculo de la notificación.

[!DNL Workfront Proof] usuarios pueden convertir archivos en pruebas después de guardarlos en su cuenta.
